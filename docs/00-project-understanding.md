# Town Diary Simulator 项目统一理解

## 1. 本文档用途

本文档基于以下三份原始材料整理：

- `小镇日记模拟器项目策划案.docx`
- `小镇日记模拟器工程实现文档.docx`
- `Codex Implementation Brief.docx`

它不是对原文的复述，而是三份材料合并后的项目定义。后续实现、审查和验收都应以本文档及同目录下的实施清单为准。

## 2. 项目本质

Town Diary Simulator 首先是一个持续运行的小镇生活模拟器，其次才是一个文本生成系统。

小镇环境拥有自己的时间、天气、地点、开放状态、居民、行为和事件。环境不依赖小说家出现才运行，也不为了产出精彩文本而临时制造剧情。

所有居民，包括小说家，都是世界中的 Agent。每个 Agent 只能根据自己能够感知的信息、个人状态、习惯、记忆和目标提出行为。行为是否真的发生，由环境统一校验和提交。

小说家的特殊性只有两点：

1. 他和其他居民一样生活、感知、行动、误解和记忆。
2. 他额外拥有记录与写作能力，能够把自己的主观经历整理成日记和小说片段。

因此，小说家不是导演、世界控制器或全知叙述者。他是一个具有主观性的 camera。

## 3. 核心因果链

项目必须保持以下单向因果关系：

1. 环境推进时间、天气和地点状态。
2. 环境为每个 Agent 生成有限 Observation。
3. Agent 根据 Observation 和自身内部状态提出 ActionProposal。
4. 环境校验、解决冲突并提交行为。
5. 已提交行为产生客观 WorldEvent。
6. 感知系统把 WorldEvent 转换成各 Agent 各自可知的 ObservedEvent。
7. Agent 更新记忆、关系认知和后续行为倾向。
8. 小说家把自己的 Observation、ObservedEvent 和 Experience 记录下来。
9. 写作系统只使用小说家可知材料生成日记、片段和章节。

任何从“想写某段剧情”反向修改世界事实的路径，都违反项目目标。

## 4. 三层现实

项目需要严格区分三层信息，不能混用。

### 4.1 客观世界层

这是环境中实际发生的事情，包括：

- 当前时间和天气。
- 地点是否开放。
- 每个居民实际位于哪里。
- Agent 提交并被环境接受的行为。
- 行为产生的客观事件及结果。
- 由环境规则推进的客观身体状态和地点状态。

客观世界层是真相，但不是任何 Agent 都能直接读取的公共信息。

### 4.2 主观认知层

这是每个 Agent 实际知道、相信或推测的内容，包括：

- 直接看到或听到的事件。
- 从他人处听来的传闻。
- 对事件的误解和解释。
- 记忆中的过去经历。
- 对其他人的熟悉、信任、紧张和猜测。

主观认知层可能不完整，也可能错误。不同 Agent 对同一 WorldEvent 可以形成不同的 ObservedEvent。

### 4.3 文本表达层

这是小说家基于自己主观认知生成的文本，包括：

- 每日观察记录。
- 每日 ExperienceLog。
- 私人日记。
- 小说化片段。
- 七天章节草稿。

文本表达层可以有文学加工，但不能增加小说家从未获得的新事实。

## 5. “Environment 独立存在”的工程含义

“环境独立运行”不只表示有一个名为 Environment 的类，而应满足以下行为：

- 即使不创建小说家，世界也能继续推进。
- 即使禁用写作模块，居民仍会行动，事件仍会产生。
- 小说家缺席的地点仍然会发生事件。
- 环境生命周期不由小说家的日记流程控制。
- 环境可以单独运行并产出 WorldLog。
- 小说家和写作系统都是环境的消费者，而不是环境存在的前提。

MVP 可以先在同一个操作系统进程内运行，但模块边界必须允许“仅运行世界”模式。是否拆成独立服务或独立进程，不是 MVP 的必要条件；运行语义独立才是必要条件。

## 6. “所有行为都是 Agent 的客观行为”的工程含义

需要区分环境变化和 Agent 行为：

- 时间推进、天气变化、地点开关属于环境规则产生的客观状态变化。
- 移动、工作、休息、观察、交谈、写笔记等属于 Agent 提议并由环境提交的客观行为。
- 对话内容、传闻传播和关系变化必须有已提交行为或已发生事件作为来源。
- 文本生成不能直接创造一个新的世界事件。
- WorldEvent 的结构化字段是事实，面向人的摘要只是事实的可读表达。

例如，“咖啡馆老板向小说家透露过去”不能作为无来源文本直接出现。它需要由同地点、交谈行为、关系状态、可透露信息和环境校验共同产生。

## 7. 小说家作为 camera 的准确边界

小说家 camera 能记录：

- 自己收到的 Observation。
- 自己直接观察到的 ObservedEvent。
- 自己听到的传闻及其低确定性。
- 自己参与的行为和行为结果。
- 自己的感受、解释、推测和误解。
- 过去已经形成的个人记忆。

小说家 camera 不能记录：

- 未在场且未听说的事件。
- WorldEvent 中未被感知系统暴露的隐藏字段。
- 其他 Agent 的真实内心状态。
- 系统为调试保存的完整因果链。
- 写作系统为了连贯性自行补充的事实。

小说家 camera 记录的是体验证据，不是世界数据库的复制。

## 8. MVP-0.1 的统一范围

### 8.1 必须包含

- 一个可独立推进的世界运行时。
- 七天模拟，每天五个 time block。
- 时间、天气、地点拓扑与开放规则。
- 五个公开叙事地点：小说家家、咖啡馆、图书馆、市场、旧车站。
- 六个 Agent：小说家与五位居民。
- 最小客观状态、需求、习惯和规则决策。
- ActionProposal、校验、冲突处理与提交。
- 结构化 WorldEvent 和 WorldLog。
- 每个 Agent 的有限 Observation。
- 小说家的 ObservationLog、ExperienceLog。
- 最小直接观察与传闻机制。
- 每日日记、每日小说片段、七天章节草稿。
- MockLLM 全流程。
- 固定 seed 的可复现运行。
- 可追踪“文本使用了哪些观察材料”的来源关系。

### 8.2 暂不包含

- Web UI、游戏引擎或完整 galgame UI。
- 真实 LLM API 作为必需依赖。
- 大规模居民、复杂经济和战斗。
- 向量数据库、复杂 RAG 和多模型调度。
- 复杂剧情导演或为了文本效果强制触发的事件。
- 长篇小说自动生成。
- 多线程或分布式模拟。

## 9. 原始文档中的冲突与统一裁决

### 9.1 天气缺少独立实施步骤

天气在策划案和工程原则中属于核心环境能力，但原实施 Brief 没有单独步骤。

统一裁决：天气系统必须在 Agent 决策和完整 Environment tick 之前完成，因为它会影响地点、移动、行为评分、观察和文本材料。

### 9.2 HabitSystem 是否属于 MVP

工程文档要求居民通过习惯权重行动，但 Brief 又把 HabitSystem 列为 MVP-0.2。

统一裁决：MVP-0.1 必须包含“最小习惯规则和评分”，否则居民只是随机移动，无法验证“像人一样基于行为习惯行动”。复杂习惯学习和动态形成推迟到后续。

### 9.3 传闻是否属于 MVP

工程文档和最终验收要求测试“传闻不是事实”，但 Brief 又把简单传闻机制列为 MVP-0.2。

统一裁决：MVP-0.1 包含最小、可测试、可追踪的传闻传递。复杂传播网络和长期失真推迟到后续。

### 9.4 ExperienceLog 是否属于 MVP

策划案与工程输出都要求 ExperienceLog，但 Brief 把它放到 MVP-0.2。

统一裁决：MVP-0.1 包含最小结构化 ExperienceLog，用规则把观察转换为感受、解释和确定性。复杂反思与 LLM 主观体验推迟到后续。

### 9.5 Agent 位置的双真源风险

原工程示例同时在 Agent 与 WorldState 中保存当前地点。

统一裁决：当前地点属于客观世界状态，只能由 Environment 持有并修改。Agent 可以通过 Observation 知道“自己当前在哪里”，但不能持有另一个可独立修改的权威位置字段。

### 9.6 居民住处引用不存在地点

原材料中的部分 `home_location_id` 没有对应地点，或者临时把工作地点当作住处。

统一裁决：实施前必须明确居民住处策略。推荐把居民私人住处建模为最小非公开地点节点，不增加公开叙事地点数量，但保证所有配置引用有效。

### 9.7 Agent 行动顺序污染

如果一个 Agent 行动后立即改变世界，再给下一个 Agent 生成 Observation，后面的 Agent 会因为遍历顺序获得不同世界。

统一裁决：每个 tick 使用两阶段流程。先基于同一世界快照为所有 Agent 生成 Observation 并收集提案，再按确定性规则统一解决和提交。

## 10. MVP 成功的真实标准

MVP 成功不以文本华丽程度为标准，而以以下性质为标准：

- 删除小说家后，世界仍能运行七天。
- 删除写作模块后，世界仍能产生完整 WorldLog。
- 同一 seed 和配置可以重放相同世界结果。
- Agent 不读取完整 WorldState。
- Agent 行为都能追溯到 Observation、习惯、需求或记忆。
- 环境拒绝不合法行为，失败行为不改变世界。
- 小说家不知道的事件不会出现在 ObservationLog、日记或章节中。
- 传闻在数据中保持低确定性，不能被写成确定事实。
- 每个文本事实都能追溯到小说家的可知材料。
- 七天章节是七天体验的整理，不是另起炉灶的小说生成。
