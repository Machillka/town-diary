# 实施路线与里程碑

## 1. 路线原则

实施顺序围绕一个原则展开：先证明世界客观存在，再证明 Agent 基于有限感知行动，最后才允许小说家写作。

每个步骤必须：

- 只实现当前步骤需要的最小范围。
- 明确数据所有权和依赖方向。
- 增加对应验证。
- 使用固定 seed。
- 保持 MockLLM 可完成全流程。
- 在退出门槛满足后再进入下一步。

## 2. 里程碑总览

| 里程碑 | 目标 | 关键证明 |
| --- | --- | --- |
| M0 项目契约 | 固化范围、数据契约和测试策略 | 团队对“世界真相、主观认知、文本输出”有一致理解 |
| M1 独立世界 | 时间、天气、地点和世界生命周期可独立运行 | 没有小说家和写作模块，世界仍能推进 |
| M2 客观行为 | Agent 提案、环境校验、事件提交可运行 | 所有居民行为都可追踪且由环境提交 |
| M3 有限感知 | 每个 Agent 获得不同的主观世界切片 | 小说家不会知道未感知事件 |
| M4 小说家 camera | 小说家记录 Observation 和 Experience | 记录来源于亲历、听闻和主观解释 |
| M5 写作交付 | 日记、片段和章节只消费允许材料 | 文本不补充小说家不知道的事实 |
| M6 七天验收 | 完整七天闭环、重放和泄漏测试通过 | MVP-0.1 可交付 |

## 3. 步骤依赖关系

### 3.1 M0 项目契约

1. 冻结 MVP 范围和术语。
2. 冻结数据所有权和依赖规则。
3. 建立项目骨架、测试入口和配置约定。
4. 定义核心数据契约和配置校验策略。

### 3.2 M1 独立世界

5. 建立确定性随机数、ID 和运行上下文。
6. 实现世界时钟。
7. 实现天气状态机。
8. 实现地点拓扑和开放规则。
9. 实现 WorldState 与只读快照。
10. 实现 World Runtime 生命周期与世界模式。
11. 实现 checkpoint、恢复和基础重放。

### 3.3 M2 客观行为

12. 实现 Agent 静态资料、主观状态和配置加载。
13. 实现最小需求、习惯和候选行为。
14. 实现 ActionProposal、校验与执行。
15. 实现结构化 WorldEvent 与 WorldLog。
16. 实现两阶段 tick 和冲突解决。
17. 实现居民规则决策和行为理由。
18. 实现最小交谈、关系认知与传闻传播来源。

### 3.4 M3 有限感知

19. 实现 Observation 与 ObservedEvent。
20. 实现直接观察、参与结果、传闻和模糊线索。
21. 实现 Agent 记忆更新和认知隔离。
22. 完成知识泄漏与权限边界测试。

### 3.5 M4 小说家 camera

23. 把小说家接入普通 Agent 决策链。
24. 实现 Recorder 和 ObservationLog。
25. 实现规则化 ExperienceLog。
26. 实现日终写作材料投影与来源链。

### 3.6 M5 写作交付

27. 实现 LLM 抽象、MockLLM 和提示约束。
28. 实现 DiaryWriter。
29. 实现 NovelFragmentWriter。
30. 实现 ChapterWriter 和章节来源清单。
31. 实现输出目录、摘要和审计报告。

### 3.7 M6 七天验收

32. 实现完整 Simulator 编排。
33. 完成七天固定 seed 集成运行。
34. 完成世界独立、有限感知、传闻和文本泄漏验收。
35. 完成重放、恢复、文档和 MVP 发布检查。

## 4. 关键路径

关键路径是：

`项目契约 -> 确定性基础 -> 时钟/天气/地点 -> World Runtime -> Action/Event -> 两阶段 tick -> Perception -> Recorder/Experience -> Writing -> 七天验收`

以下能力不能提前：

- Perception 未完成前，不能把 WorldLog 当作小说家材料临时替代。
- Recorder 未完成前，不能让 DiaryWriter 直接读取 WorldLog。
- 来源链未完成前，不能把章节“看起来合理”当作验收通过。
- 世界模式未完成前，不能宣称环境能够独立运行。

## 5. 每个里程碑的交付门槛

### M0 退出门槛

- 所有术语有唯一含义。
- 数据所有权无冲突。
- MVP 包含项和延期项已明确。
- 测试和输出约定已明确。

### M1 退出门槛

- 无 Agent 时可推进至少七天。
- 时间、天气、地点状态按规则变化。
- 同一 seed 的世界系统变化可复现。
- checkpoint 可保存并恢复。

### M2 退出门槛

- 六个 Agent 可加载。
- Agent 只能提出行为，不能直接修改世界。
- 所有成功行为产生结构化 WorldEvent。
- 所有失败行为不改变世界。
- Agent 遍历顺序不影响同一 tick 的决策输入。

### M3 退出门槛

- 不同地点 Agent 得到不同 Observation。
- 未传播的异地事件不会进入小说家认知。
- 传闻保留来源和低确定性。
- 权限与泄漏测试通过。

### M4 退出门槛

- 小说家走普通 Agent 路径。
- Recorder 只记录小说家实际收到的材料。
- Experience 可以追溯到 Observation 或 ObservedEvent。
- 日终材料不包含完整 WorldLog。

### M5 退出门槛

- 无真实 API 时可生成全部文本产物。
- 每日日记只使用当日允许材料。
- 小说片段不添加新事实。
- 章节可追溯到七天日记与片段。

### M6 退出门槛

- 完整七天运行成功。
- 世界模式、观察模式和完整模式均可运行。
- 固定 seed 重放一致。
- 核心泄漏场景全部通过。
- 交付目录和审计材料完整。

## 6. 风险优先级

### 最高风险

- Agent 或 Writing 直接读取 WorldState / WorldLog。
- Agent 直接修改世界。
- 事件只有自然语言摘要，没有结构化事实。
- 行动顺序造成隐式信息泄漏。
- 传闻无传播来源，实际变成全知信息。
- 章节生成补充了小说家不知道的真相。

### 中等风险

- 配置引用不存在的地点或 Agent。
- 当前地点、需求或关系出现双真源。
- seed 只控制部分随机数，无法重放。
- MockLLM 输出与输入无关，导致测试无法验证来源约束。

### 后期风险

- 真实 LLM 非确定性。
- 文本质量不足。
- 更复杂的关系、记忆和事件节奏。

## 7. 延期到 MVP-0.2 的内容

- 动态形成和学习习惯。
- 多跳传闻、传闻失真和遗忘。
- 双向且非对称的复杂关系模型。
- 长期记忆检索和反思。
- 真实 LLM 可选接入。
- 更复杂的情绪、写作压力和灵感循环。
- 每三天小结和角色事件阶段。
- 可视化和交互界面。
