## 代码解释

组合 checkpoint 保存 RunContext manifest、WorldStateCheckpoint、已提交 tick 记录、生命周期状态和 schema version。RunContext manifest 已包含 seed、配置摘要、随机数状态和 ID 生成器状态，因此恢复后的随机序列可以无缝延续。

未结束 runtime 恢复后固定为 `paused`，调用方必须显式恢复运行。重放不会使用 checkpoint 中的 WorldState 作为起点，而是用原始 seed 和配置从头运行到 checkpoint tick。

checkpoint 文件使用独占创建模式，拒绝覆盖已有文件。

## 关键分析与取舍

- Step 15 才会建立完整 WorldLog；本步骤使用确定性的 `WorldTickRecord` 验证客观环境结果一致性。
- 配置定义本身不重复保存，但必须通过 SHA-256 配置摘要验证为同一份配置。
- 恢复路径只读取结构化状态，运行摘要和未来文学文本都不能成为状态来源。
