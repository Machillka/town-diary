## 契约解释

架构契约通过“唯一所有者、允许读取者、允许修改者”约束每类数据。

最重要的规则：

- Environment 独占客观状态修改权。
- Agent 只读取 Observation，只能提出 ActionProposal。
- Perception 是 WorldEvent 到 ObservedEvent 的唯一转换入口。
- Recorder 只记录小说家实际收到的材料。
- Writing 只读小说家材料，不能读取 WorldLog 或 WorldState。
- app 层只组装模块，不能绕过 Environment。

## 关键分析与取舍

- Agent 当前地点只存在于 WorldState；Agent 从 Observation 得知自己在哪里。
- tick 采用“统一快照 -> 收集全部提案 -> 统一提交”的两阶段流程，避免遍历顺序污染。
- 定义 `world`、`observe`、`full` 三种运行语义，用于证明环境独立存在。
- 自然语言摘要不能成为规则判断的唯一事实来源，后续事件必须结构化。
