# Step 17：居民规则决策与行为理由

状态：Completed  
日期：2026-06-14

## 范围与退出门槛

本步骤让 Agent 在两阶段 tick 中选择最高分合法候选，并提供可审计理由、明确 fallback 和七天确定性行为闭环。不调用真实 LLM。

退出门槛：

- 六个 Agent 每 tick 都提出行为。
- 决策保存完整候选、选择结果和理由。
- 行为与习惯、需求、天气、地点和关系相关。
- 无候选时安全 fallback。
- 七天行为与 WorldLog 可按 seed 重放。

## 代码解释

RuleDecisionPolicy 调用普通 Agent 的候选生成路径，选择已按分数排序的第一项并生成 ActionProposal。每次选择保存 DecisionAudit；行为理由只留在审计和提案中，不写入客观 WorldEvent facts。

若没有候选，规则明确生成 stay fallback。DecisionAudit 支持 checkpoint/restore，失败 tick 不会留下未提交决策记录。RuleBasedTownSimulation 组合 Agent、RuleDecisionPolicy 和 TickCoordinator，同时复用 WorldRuntime 的状态、上下文、WorldLog、生命周期和摘要。

world 模式从正式配置中排除小说家，五位居民仍使用同样规则行动，不创建 Recorder 或 Writer。

## 关键分析与取舍

- 疲劳通过降低 work/move 分数而不是直接禁止行为，让休息成为可解释的最高分选择。
- 晚间返家是稳定生活节奏规则，不是剧情导演。
- 当前规则选择最高分候选；冲突后失败不会在同一 tick 再选第二候选，避免隐式单阶段重决策。
- 复杂习惯学习、动态关系和 LLM 决策仍明确延期。
