## 代码解释

### 所有权

`WorldClock` 是可变对象，因为世界时间需要推进；按照架构契约，它只能由未来 Environment 持有和调用 `advance`。

其他模块应使用不可变 `ClockSnapshot`，不能获得推进能力。

### 日终与周终语义

- 当前时间为 `night` 时，`is_day_end()` 为真。
- 当前 day 是七的倍数且时间为 `night` 时，`is_week_end()` 为真。
- 第七天 night 是完整七天的最后一个 tick；再推进一次进入第八天 morning。

### 快照恢复

`ClockSnapshot` 保存 day、time block 和 schema version，可转换为普通字典，并在恢复时校验字段、枚举和版本。

## 关键分析与取舍

- `TimeBlock` 定义在 core contracts，确保 Observation、WorldEvent 和 WorldClock 使用同一枚举。
- 时钟不读取 RunContext，也不含随机性。
- 没有把时钟接入 bootstrap Simulator，避免提前实现 Environment 生命周期。
