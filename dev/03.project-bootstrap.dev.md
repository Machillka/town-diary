## 代码解释

### `town_diary.main`

这是稳定的公共 CLI 入口，只把参数传给 `app.cli.run_cli`。它不持有领域规则。

### `app.cli`

CLI 负责参数解析和最小输入校验，然后构造不可变的 `SimulationRequest`。`mode` 参数提前体现架构契约，但当前不实现模式行为。

### `app.simulator`

`Simulator` 是应用编排边界的占位实现。它只报告请求，不创建输出目录、不推进世界，也不调用未来模块。这个限制防止 Step 03 偷跑后续功能。

### 包级 `__init__.py`

这些文件只建立模块所有权边界，不包含类、接口或领域逻辑。

## 关键分析与取舍

- 使用标准库 `argparse`，避免为最小 CLI 引入额外框架。
- `SimulationRequest` 使用冻结 dataclass，防止参数在编排过程中被意外修改。
- CLI 当前不创建输出目录，避免在尚未定义覆盖策略和 run manifest 前产生副作用。
- 项目声明 Python 3.11+；本次 uv 验证实际使用 CPython 3.14.5，属于支持范围。
- MockLLM 目前只作为允许的 CLI 选项，未实现任何生成逻辑。
