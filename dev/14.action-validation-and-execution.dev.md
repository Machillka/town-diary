# Step 14：ActionProposal、ActionValidator 与 ActionExecutor

状态：Completed  
日期：2026-06-14

## 范围与退出门槛

本步骤建立“Agent 只能提议，Environment 决定是否发生”的行为边界，支持 stay、move、work、rest、observe、talk 和 write_notes。不实现 WorldLog；完整 WorldEvent 生成属于 Step 15。

退出门槛：

- Agent 只能把候选转换为 ActionProposal。
- 每类行为都有明确校验规则和客观效果。
- 未知行为和所有非法行为明确失败。
- 失败行为不修改 WorldState。
- 成功行为只经 ActionExecutor 修改 WorldState，并返回结构化 ActionResult。

## 改动

- 扩展 `ActionResult`，增加结构化 `effects`。

## 代码解释

`ActionValidator` 只读检查当前 WorldState 和正式 Agent 配置：

- stay、rest、observe 不接受目标。
- move 要求目标存在、相邻、开放；私人地点仅允许对应 Agent 的住处。
- work 要求居民处于当前时间激活的习惯地点，且地点开放。
- talk 要求目标人物存在、不是自己且与提案者同地点。
- write_notes 只允许小说家。
- move、work、observe、talk、write_notes 要求 Agent 仍有能量。
- 未知行为类型直接拒绝。

`ActionExecutor` 先完整校验，失败时直接返回空 effects；成功时通过 WorldState 明确修改入口提交。move 修改客观位置，work/rest/talk 修改客观身体状态，stay/observe/write_notes 当前不产生客观状态变化。

## 关键分析与取舍

- ActionProposal 仍然只是意图，Agent 的 `propose` 方法没有执行能力。
- Step 14 的 ActionResult 使用 `effects` 表示结构化提交结果，`events` 保持为空；Step 15 将把已提交结果转换成 WorldEvent。
- 当前每个成功行为最多执行一个 WorldState 修改操作，因此校验完成后不会留下部分提交；多行为冲突和 tick 级原子提交属于 Step 16。
- 行为效果把能量和饥饿限制在 `0..100`，避免行为执行制造非法数值。
