## 代码解释

### 唯一客观真源

WorldState 持有：

- WorldClock。
- WeatherSystem。
- LocationSystem。
- Agent 客观位置。
- Agent 客观身体状态。
- tick 序号。

当前没有 Agent 类，因此不存在 Agent 内部当前位置。后续 Agent 只能从 Observation 得知位置。

### 修改边界

WorldState 提供明确的客观修改方法：

- `move_agent`
- `set_body_state`
- `increment_tick`

所有方法先验证候选值，再提交修改。失败修改不会留下非法中间状态。

### 深度不可变快照

`snapshot()` 复制当前客观状态到冻结 dataclass 与 tuple 中。快照：

- 没有修改方法。
- 后续 WorldState 变化不会反向改变旧快照。
- 仍然属于完整客观投影，只允许未来 Environment 和 Perception 使用。

### Checkpoint

WorldStateCheckpoint 可转为 JSON 兼容字典，保存时钟、天气、Agent 状态和 tick。它不保存随机数状态；完整恢复必须同时使用 RunContext manifest。

## 关键分析与取舍

- WorldState 暂时公开 `clock` 和 `weather` 作为 Environment-only 能力；Agent 永远不会获得 WorldState 引用。
- LocationSystem 是冻结拓扑，因此 checkpoint 通过正式配置重建，不重复保存地点定义。
- 默认身体状态仅用于建立客观状态容器，正式需求配置属于 Step 12-13。
- checkpoint 实现提前服务 Step 09 的恢复一致性，完整运行 checkpoint 编排仍属于 Step 11。
