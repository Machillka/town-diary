状态：Completed  
日期：2026-06-12

## 范围与退出门槛

本步骤定义模块交换数据、schema 版本和配置校验，不实现正式小镇配置、Environment 或 Agent 行为。

退出门槛：

- 核心数据契约字段和信息边界明确。
- ID 格式稳定。
- 配置缺失、重复 ID、无效引用和非法枚举会被拒绝。
- 一组有效最小配置可以加载。

## 代码解释

### 不可变契约

`WorldSnapshot`、`Observation`、`ActionProposal`、`ActionResult`、`WorldEvent`、`ObservedEvent` 和 `Experience` 使用冻结 dataclass，避免跨模块传递后被接收方静默修改。

### 结构化事实与可见性

`WorldEvent.summary` 仅用于人类阅读。规则和感知应使用 `EventFact`；每个 Fact 明确标记 `public`、`participant` 或 `hidden`，为后续 Perception 过滤建立边界。

### 配置校验

加载和校验分离：

- `load_config_bundle` 负责安全读取三个 YAML 文档。
- `validate_config_documents` 负责 schema 和引用完整性。

测试使用临时最小配置，不提前创建 Step 08 的正式地点配置。

## 关键分析与取舍

- 引入成熟的 PyYAML，只用于安全 YAML 解析；不引入配置框架。
- ID 使用 NewType 提供静态语义，运行时通过显式函数校验格式。
- 核心契约只表达最小字段，复杂记忆、关系和写作材料留给后续步骤。
- `WorldSnapshot` 明确是完整客观只读投影，Agent 仍不得读取它。
