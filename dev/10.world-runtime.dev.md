## 代码解释

`WorldRuntime` 是 Environment 生命周期入口，持有 `RunContext` 和 `WorldState`。每个 tick 只推进天气、tick 序号和时间，不创建任何 Agent 实例，也不加载 Recorder 或 Writing。

生命周期为：

`created -> running -> paused -> running -> ended`

无效状态转换会抛出 `RuntimeLifecycleError`。运行结束必须记录 `completed` 或 `stopped` 原因。

`WorldTickRecord` 是 Step 10 的环境运行记录，不等同于 Step 15 的完整 `WorldLog`。它只证明环境变化具有确定、结构化、可比较的输出。

## 关键分析与取舍

- WorldState 已包含配置中的客观 Agent 位置，但 world 模式不会创建 Agent 行为对象；这保持了客观世界配置完整，同时满足“删除小说家后世界仍能运行”。
- `observe` 和 `full` 暂时保留 bootstrap 行为，避免提前引入 Recorder、Writing 或尚未实现的行为系统。
- tick 记录发生在当前时间块提交后，随后时钟推进到下一时间块。
