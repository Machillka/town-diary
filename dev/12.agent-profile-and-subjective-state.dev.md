## 代码解释

小说家和居民全部由同一个 `Agent.from_config` 路径创建，小说家仅由 `role == "novelist"` 标记。Agent 静态身份、profile、habits 和 goals 冻结不可修改。

每个 Agent 内部独占一个 `SubjectiveState`，用于 mood、记忆和关系认知。外部只能取得冻结快照，不能取得内部容器。

Agent 不保存 `location_id`、`body_state` 或 `WorldState`。读取客观位置和身体状态时，必须传入 observer ID 与自身一致的 `Observation`。

## 关键分析与取舍

- Step 12 只建立习惯和目标资料，具体需求、触发、候选行为与评分留给 Step 13。
- `home_location_id` 是静态身份资料，不是当前客观位置。
- 关系是单个 Agent 的主观认知，不自动对称。
- 初始记忆使用简短文本容器；结构化事件记忆将在后续记忆步骤扩展。
