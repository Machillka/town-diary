## 契约解释

`mvp-scope.md` 将实现目标压缩为六项可证明能力：

1. 世界可独立运行。
2. Agent 基于有限感知提出行为。
3. Environment 提交客观事实。
4. 小说家只记录自身体验。
5. Writing 只使用小说家材料。
6. 七天材料形成章节草稿。

术语表明确了 Environment、WorldState、Observation、ActionProposal、WorldEvent、ObservedEvent、Experience、Recorder 和 Writing 的唯一含义，避免实现时把意图、事实、认知和文本混为一体。

## 关键分析与取舍

- 采用“最小非公开私人住处节点”，而不是把工作地点当住处或使用无空间语义的 offstage 状态。
- 最小 Habit 是“居民像人一样按习惯行动”的必要条件，属于 MVP。
- 最小传闻是验证“传闻不是事实”的必要条件，属于 MVP。
- 规则化 ExperienceLog 是小说家主观体验进入写作材料的必要桥梁，属于 MVP。
- 真实 LLM、复杂关系、复杂记忆和 UI 延期，避免分散核心闭环。
