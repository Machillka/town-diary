## 代码解释

TickCoordinator 在环境变化后创建统一 WorldSnapshot，并为所有 Agent 构建 Observation。Agent 按稳定 ID 顺序生成提案；全部提案收集完成后，协调器检查 WorldState 未变化，再统一校验和解决冲突。

冲突资源至少包含行为发起者；talk 同时占用目标人物。有效提案按稳定 action ID 获取 seed 控制的优先级，赢家之后按 action ID 稳定提交。

若任何阶段抛出异常，协调器恢复 tick 开始时的 WorldStateCheckpoint、随机数状态、ID 计数器、WorldLog 长度和可 checkpoint 的决策策略审计。

## 关键分析与取舍

- 当前地点容量无限，因此 move 不以目标地点作为冲突资源。
- 当前 ObservationBuilder 只暴露同地点人物、自己状态和合法相邻地点，不暴露完整 WorldState。
- Agent 主观状态回滚将在后续主观状态 checkpoint 步骤扩展；当前规则提案阶段不会修改主观状态，DecisionAudit 已纳入事务。
- TickReport 保存共同快照、Observations、Proposals、Results、Events 和环境记录，便于审计。
