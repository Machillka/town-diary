# Step 13：最小需求、习惯与候选行为生成

状态：Completed  
日期：2026-06-14

## 范围与退出门槛

本步骤只从 Agent 静态/主观状态、Observation 和注入的确定性随机源生成候选行为，不提交 ActionProposal，也不修改 WorldState。

退出门槛：

- 每个 Agent 至少有一项稳定习惯。
- 候选行为只使用有限 Observation 和 Agent 自身资料。
- 每个候选包含来源、评分贡献和可读理由。
- 随机扰动只使用 `DeterministicRandom`。
- 不生成当前 Observation 标记为不可用的行为、地点或人物目标。

## 代码解释

`NeedState` 从 Agent 自己的 Observation 派生能量恢复、饥饿、社交和小说家写作倾向，不保存第二份客观状态。

`CandidateGenerator` 仅使用：

- Observation 中的时间、天气、身体状态、可见人物、可用行为和可达地点。
- Agent 自己的习惯、目标、role 和主观状态快照。
- 调用方注入的 `DeterministicRandom`。

候选行为包括 stay、move、work、rest、observe、talk 和 write_notes。每个候选保存 habit、need、weather、goal、relationship、subjective_state 和 random 等评分贡献；候选仍是主观选项，不是 ActionProposal 或客观事实。

## 关键分析与取舍

- `available_location_ids` 是有限 Observation 投影，不是完整地点拓扑；它防止 Agent 自行读取 WorldState 来判断移动目标。
- 饥饿需求已被派生，但 MVP 行为集合尚无 eat 行为，因此本步骤不让它产生虚假的进食候选。
- 小说家使用普通 Agent 的同一生成器，只通过 role、习惯和写作倾向产生差异。
- 候选按分数和稳定字段排序，随机扰动幅度小且完全受统一 seed 控制。
